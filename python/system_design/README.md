# System Design

Complete system design exercises covering caching, load balancing, rate limiting, 
sharding, message queues, and complete system designs.

## Exercises

1. **ex1_caching_strategies** - LRU cache, write-through, TTL cache
2. **ex2_load_balancing** - Round-robin, least connections, weighted, IP hash
3. **ex3_rate_limiting** - Token bucket, sliding window rate limiters
4. **ex4_sharding** - Consistent hashing, range-based sharding
5. **ex5_message_queue** - Simple queue, pub/sub pattern
6. **ex6_system_design** - URL shortener, key-value store
